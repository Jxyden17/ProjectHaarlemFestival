# Stories State Diagram

This state diagram describes the lifecycle of a Stories booking from browsing a session through payment and ticket handling.

```mermaid
    [*] --> BrowsingStories

    BrowsingStories --> SessionSelected : user selects story session
    SessionSelected --> TicketPending : booking started

    TicketPending --> PaymentInProgress : user proceeds to checkout
    PaymentInProgress --> PaymentFailed : payment rejected or cancelled
    PaymentInProgress --> PaymentCompleted : payment accepted

    PaymentFailed --> TicketPending : retry payment
    TicketPending --> Cancelled : user abandons booking

    PaymentCompleted --> TicketIssued : ticket generated
    TicketIssued --> CheckedIn : ticket scanned at event
    TicketIssued --> Refunded : admin refund
    TicketIssued --> Expired : event date passed

    CheckedIn --> [*]
    Cancelled --> [*]
    Refunded --> [*]
    Expired --> [*]
```
